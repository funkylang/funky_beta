std_types::io/std::set_timeout

sets the timeout for the next call to std::get_events

Parameters

self
the input-output object
timeout
the timeout in seconds

Result

io
the updated input-output object

Description

The timeout is reset to std::undefined after each call to std::get_events.

If the timeout is std::undefined then std::get_events will wait indefinitely for an event to occur.

If the timeout is a number then std::get_events will wait for at most this number of seconds for an event to occur.

If the timeout is a std::date_and_time object then std::get_events will wait until this time for an event to occur.

If the timeout is a negative number then std::get_events will return immediately if no event occurred.
