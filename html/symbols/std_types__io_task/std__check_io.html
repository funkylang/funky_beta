std_types::io_task/std::check_io

checks the result of an I/O-operation

Parameters

self
the task that initiated the I/O-operation

argument

Result

task
the task that initiated the I/O-operation

Description

This function is called by the I/O-manager to check the result of an I/O-operation.

The function std::body is called with the following arguments:

  • the task that initiated the I/O-operation
  • the list of I/O-events that occurred since the last call to std::check_io
  • the argument passed to std::check_io

The function std::body must return a tuple consisting of

  • the new mode
  • the task that initiated the I/O-operation
  • the I/O-object
  • the body function
  • the argument to pass to the body function

The following modes are defined:

  • std::COMPUTE: the task is in compute mode
  • std::WAIT: the task is waiting for an event
  • std::START_IO: the task is in start-I/O-mode
  • std::CHECK_IO: the task is in check-I/O-mode
  • std::COMPLETE_IO: the task is in complete-I/O-mode
  • std::EXIT: the task is in exit-mode
